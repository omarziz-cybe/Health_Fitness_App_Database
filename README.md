# Health & Fitness App Database System 🏋️‍♂️📊

## 📌 Project Overview
A comprehensive relational database designed and implemented to support the backend operations of a Health and Fitness tracking application. This project outlines the core data architecture required to efficiently manage user profiles, workout routines, dietary plans, and fitness progress tracking.

This project demonstrates strong skills in database design, data modeling, and writing optimized SQL queries for real-world applications.

## 🚀 Features
- **Robust Relational Design:** Structured with normalized tables (e.g., `Users`, `Workouts`, `Exercises`, `Diet_Plans`, `Progress`) to maintain data integrity and eliminate redundancy.
- **Complex Queries & Views:** Includes advanced SQL queries, multi-table JOINs, and views to generate detailed user progress reports and calculate daily caloric intake.
- **Data Integrity & Security:** Implemented strict Primary Keys (PK), Foreign Keys (FK), and constraints to ensure robust and error-free data relationships.
- **Scalable Architecture:** Designed to easily accommodate future features like premium subscriptions or social interactions between users.
